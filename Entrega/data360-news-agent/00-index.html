Data360 News Agent

AI-powered news agency that detects newsworthy facts from Data360's 12,000 indicators and delivers them verified with local perspective to LAC newsrooms — operationalizing information integrity through authoritative sourcing, source traceability, and AI-readiness.

A prototype by Abrimos.info for the Data360 Global Challenge (World Bank and Media Party), phase 2.

Architecture Overview

Data360 News Agent makes an explicit distinction between reused battle-tested infrastructure (agentic LLM framework, newsletter distribution layer, TeseoETL ingestion core — in production since 2019) and substantial new development: five novel detection strategies for development indicators, a Fact-Checker Agent with veto power over every numeric claim, and a citizen-narrative module (Noticia + Reportaje) built from scratch for this prototype. The system operationalises Craig Hammer's criteria — data reliability, trustworthiness, and discoverability — through Proof-Carrying Numbers (PCN): every published figure is bound to a verifiable {indicator, country, date, value, endpoint} tuple and resolves back to the Data360 source in two clicks. Interpretability is a precondition for emission: the Fact-Checker Agent declares data gaps rather than imputing values, in explicit alignment with IFCN principles of transparency of methodology. The pipeline augments newsrooms — it does not replace editors: every alert surfaces as a reviewable draft, with partner editors retaining full editorial override before publication. Open-source models (Llama, Mistral, Qwen) are prioritised over closed APIs by design; the stack is self-hostable end to end; and the connector is country-agnostic — the five LAC countries in the demo are a focus choice, not an architectural constraint on a system ready to serve any of Data360's 200+ countries. Code: GPLv3.

Data360 API Integration Methodology

The Data360 integration is continuous and dynamic — never static snapshots: a conditional HEAD probe against each indicator's bulk CSV blob detects upstream changes on a scheduled cadence, and only changed indicators are selectively fetched, so every generated story is independently rebuildable from the API alone using indicator code, country code, date, value, and endpoint. This approach follows Craig Hammer's own framing of Data360 as a knowledge bank whose critical properties are data quality, authoritative sourcing, AI-readiness, and discoverability: Data360 is the primary and authoritative anchor of every published claim, and no intermediate cache substitutes for the source. Six endpoints are documented with their OData conventions and a reproducibility contract — any fact-checker, editor, or policymaker can replay the original API call and confirm or refute any figure. When an indicator does not exist for a given country-year, the system declares the gap explicitly; it does not interpolate or infer. The reference implementation is GPLv3, ensuring that this discoverability layer for the World Bank's knowledge bank remains a public good for newsrooms and institutions across the Global South.

Security & Data Handling Protocols

Data360 News Agent is self-hosted by default — data sovereignty and cost predictability for public-interest newsrooms in the Global South are built-in design constraints, not afterthoughts — and grounded in battle-tested foundations: TeseoETL has been ingesting government data sources in production since 2019. The security protocol covers the full pipeline: public indicator data from Data360 (no personally identifiable information) is ingested dynamically with license references preserved in alert metadata; subscriber emails are stored in an append-only, gitignored file on the server filesystem, with a production roadmap for encrypted storage and one-click unsubscribe; chat input lives in browser sessionStorage only, never persisted server-side. LLM provider choice is configurable to avoid vendor lock-in: open-source models (Llama, Mistral, Qwen via LAIA/vLLM) are prioritised over closed APIs, and operators in any regulatory jurisdiction — EU, Brazil, Singapore — can run the entire stack against self-hosted open-weight models with no functional loss. Information integrity is operationalised, not invoked: the PCN verification pipeline drops any alert with an unverified claim before it reaches the published feed, and every numeric figure is independently verifiable in two clicks. Source code: GPLv3 — a deliberate choice ensuring the core verification layer stays freely accessible to public-interest media in the Global South.

4. User Guide / Documentation

The user guide is written for the editor of a Latin American or Caribbean newsroom, not for a software team: it explains how to navigate the country front pages (newspaper layout with featured Reportaje and indicator headlines), how to verify PCN claim tokens against Data360 before publication, and how to use the global analytical chat to explore the full 12,000-indicator catalogue. The system produces two verified content types — Noticias (bilingual, 250–600 words, per indicator and country) and Reportajes (bilingual, 500–1,200 words, regional syntheses per dataset) — each carrying claim tokens that link every numeric figure to its authoritative Data360 source in two clicks, making data reliability and source traceability user-facing features, not backend properties. The editorial workflow is explicit: the system suggests, the newsroom decides — alerts surface as reviewable drafts; the editor verifies claim tokens, adds local context, and makes the final publication decision; AI augments the journalist, it does not replace the editor. Day-one distribution partners — Animal Político, Quinto Elemento Lab, and Ojoconmipisto — are integrated as readers of the alert feed, not as a logo wall. The guide is available in English and in Spanish.

5. Sustainability / Maintenance Plan

The sustainability plan presents Data360 News Agent as both a public good for public-interest media in the Global South and a B2B software product for data-publishing institutions — supported by three primary revenue lines: institutional implementation contracts (operated or enabled deployments for multilaterals, regional commissions, and national statistical offices); bilateral and foundation grants (the active IPI-funded Guatemalan procurement extension is the live precedent, proving the pipeline generalises beyond Data360 to other authoritative indicator catalogues); and premium editorial services for tier-1 LAC newsrooms. Unit economics are anchored in an observed reference run: steady-state daily cost on paid APIs runs approximately $5–6 for analysis, with a documented zero-cost fallback path (LAIA + NVIDIA NIM + Gemini Flash-Lite) that keeps the free tier viable for newsrooms in developing countries independent of any commercial contract. Editorial independence is a design property: GPLv3 ensures every derivative remains a public good; the PCN verification trace makes any published claim independently reproducible without going through Abrimos or its funders; and the receiving newsroom retains full editorial override over every generated story — the funder funds the pipeline, the newsroom owns the story. The roadmap milestones — SMTP newsletter, NiFi + OpenSearch scheduling, full activation of all five detection strategies, self-hosted LLM packaging — move the demo toward production without architectural refactor, and the connector's country-agnostic design means the same architecture is ready for any of the 200+ countries covered by Data360.
